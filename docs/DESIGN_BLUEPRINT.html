3 · The master prompt

Paste the block below into Figma AI (or any design tool). For tools with input limits, feed Foundations + Components first, then one screen section at a time.

You are designing Vita, a mobile AI wellness & fitness coach app (iOS + Android, 390×844 baseline). Produce a complete, polished, production-quality UI kit and all screens below as one cohesive design system. Follow this brief exactly.

Product. Vita turns a friendly 20-step onboarding into a personalized plan: daily calorie/macro targets, a weekly workout program with a guided player, food logging by photo or text, water + step + mood tracking, weather-aware hydration advice, and an AI coach chat. Tone: calm, warm, encouraging, trustworthy — a supportive coach, never a shouty fitness app. Audience: everyday adults building a wellness routine.

Design language. Two contexts, one family: Onboarding is light & airy (welcoming first impression); the main app is dark + sage (focused, premium). Soft, rounded, generous spacing; large rounded corners; soft shadows. No neon, no harsh gradients. Nature-inspired sage palette. Friendly micro-personality: emoji "characters", gentle motion, encouraging microcopy.

Color — brand: sage #6B9080 (primary), deepSage #2E3D38 (text on light), sageLight #9CC3B2 (accent on dark), softBackground #F6F8F7 (onboarding bg). Dark app surfaces: bg #14201B, surface #1E2C26, surfaceAlt #263A32; text white / white-70% / white-38–54%. Accents: blue #5B8DB8, orange #E39A53, pink #D16BA5, purple #7E6CC4, green #6BA368, red #E0566E. Semantic: macros P #6B9080 / C #5B8DB8 / F #D9A86C; rings Calories #6B9080 / Water #5B8DB8 / Steps #E39A53; categories Strength #6B9080, Lower #5B8DB8, Core #E39A53, Cardio #E0566E, Mobility #7E6CC4.

Type: humanist sans (Inter / SF Pro / Plus Jakarta Sans). Display 24–26/w800 · Title 16–18/w700–800 · Body 14–15/w400–500 · Caption 11–12/muted; body line-height ~1.35. Spacing: 4/8/12/16/20/24; screen padding 20; card padding 16–22. Radius: pills/chips 20–24 · cards 18–24 · inputs 14–16 · sheets 24 top. Elevation: flat default; hero cards one soft colored shadow (sage @30%, blur 24, y+12); dark cards use 1px white-6% borders. Icons: rounded Material. Mood scale: emoji 😞 😕 😐 🙂 😄.

Components (build the library first): Buttons (filled sage r14 / white-on-color hero / tonal / outlined / icon / FAB.extended "Log meal"); Stat ring (circular progress, stroke 7, rounded cap, center value+unit, icon+label below) in Calories/Water/Steps; Cards (plain / gradient hero / info card with emoji+title+body / weather card with left temp column + advice); Chips (🔥 streak, WORK/REST phase, single & multi select, week day-card with ✓); Segmented control (Photo/Describe); Macro pill (grams + P/C/F letter, macro-tinted); Donut chart (3 macro segments, "kcal left" center); Chat bubbles (assistant left/darkSurface, user right/sage, + typing bubble); Composer (pill field + circular send); Inputs (text, slider 1–10 with value, measurement field with cm/ft & kg/lb toggle, time picker, DOB picker); Bottom nav (Home/Coach/Plan, dark, sage indicator, center FAB on Home); Banners ("not medical advice" + crisis resources); Animated exercise demo (square: concentric pulsing rings + bobbing category icon + category-label pill, tinted per category).

States per interactive screen: Empty · Loading · Default · Error · Success/Done · Disabled. Onboarding = light, main app = dark.

Screens to deliver: (1) Onboarding — a reusable step template (progress bar, character hero, animated blobs, question, input, Back+Continue) plus variants: single-select, multi-select chips, slider, measurement+unit-toggle, time picker, and final "Save my plan". (2) Plan-generating loading. (3) Dashboard — header+streak, mood check-in + acknowledged state, Today rings + water controls, weather/hydration card, workout gradient hero, week strip with ✓, macro donut + targets, focus & mind cards, disclaimer. (4) Coach chat — header, both bubble types, typing state, composer, first-message empty state. (5) Plan — targets, expandable workout week, sample meals. (6) Log a meal — Photo & Describe modes, analyzing state, editable result card, error. (7) Workout Player — WORK and REST variants (different accent), animated demo, timer ring, controls, + completion screen. (8) Bottom-nav shell showing all 3 tabs with the FAB.

Accessibility & trust: WCAG-AA contrast on dark; tap targets ≥ 44px; never color-only (pair icon/label); persistent "not medical advice" footer; calm, non-judgmental copy.

Output: a structured Figma file — Page 1 Foundations (colors as variables, type styles, spacing, elevation), Page 2 Components (library with variants & states), Page 3 Onboarding (light), Page 4 Main App (Dashboard, Coach, Plan, Log meal, Workout Player — dark), each screen on a 390×844 frame, named clearly, using Auto Layout and shared variables for every token.
